# Phase 1-3: 営業／顧客ヒアリング雛形（人間タスク）

> Issue #90 / 作成日: 2026-05-04 / 担当: Claude (BEERACLE-CHAREVIE)

**重要**: 本ノートは Claude が代替実施できない **人間タスクの雛形**。Phase 2（仕様書改定）後、もしくは本 PR マージ後に営業／商品担当が以下の項目で実施し、結果が確定値の見直しを示唆した場合は別途 Issue を起票して再リファクタリングを行う。

---

## 1. 既存顧客ヒアリング（5〜10 社）

### 1.1 対象選定基準
- 過去 6 ヶ月以内に診断 PDF を商談で提示した顧客
- 業種ミックス（IT・製造・小売・サービスから各 1〜2 社）
- 企業規模ミックス（30〜99 人 / 100〜299 人 / 300 人以上）

### 1.2 ヒアリング項目
1. **「3 年間トータルインパクト」の数字に対する体感**
   - □ 過大（信用しがたい大きさ） / □ 適正 / □ 過小（もっとあるはず）
   - 体感の根拠（自社実績との乖離など）
2. **自社の実態数値との乖離**
   - 実態時給（社員平均、社保込み）: ___ 円
   - 月稼働日数（年間休日／有給取得実績）: ___ 日
   - 改修頻度（過去 1 年の Web/業務システム改修回数）: ___ 回
3. **5 段階内製化選択肢の選びやすさ**
   - □ ちょうど良い / □ 中間値が欲しい（4 段階／7 段階を望むか）
4. **商談時の説明可否**
   - □ 数字の根拠を聞かれた / □ 聞かれていない
   - 聞かれた場合の質問内容: ________________

## 2. 商談未成約顧客ヒアリング（5〜10 社）

### 2.1 対象選定基準
- 過去 3 ヶ月以内に商談したが成約に至らなかった顧客
- 「数字に納得感がなかった」「経営判断材料として弱かった」と推察される顧客優先

### 2.2 ヒアリング項目
1. **数字の信頼性が成約判断に与えた影響**
   - □ 大きく影響した / □ 多少影響した / □ 影響しなかった
2. **どの数字に違和感があったか**
   - □ 月額ベンダー費用の前提 / □ 改修頻度 / □ 時給 / □ 手作業時間 / □ 内製化係数 / □ 全体的な ROI
3. **公的統計出典が記載されていれば成約判断に有利だったか**
   - □ Yes / □ No / □ どちらでもない

## 3. 営業チームヒアリング

### 3.1 質問項目
1. **商談で「この数字の根拠は？」と問われた頻度**
   - □ 毎回 / □ 半数以上 / □ 半数未満 / □ ほぼ問われない
2. **問われた際の回答内容**
   - 現状の説明ロジック（仕様書 §3）で対応可能か
   - 公的統計出典付きの説明（本 PR で追加予定）が有効か
3. **数字の信頼性と成約率の相関感**
   - 数字の信頼性が高いほど成約率が上がる体感の有無

## 4. ヒアリング結果のフィードバックループ

### 4.1 結果が示す可能性のあるアクション
| ヒアリング結果のパターン | 想定アクション |
|---|---|
| 「現行値で適切」が多数 | **現状維持**。本 PR の変更で十分 |
| 「過大」の声が多い | 業種別係数 / 時間価値割引の導入（別 Issue） |
| 「過小」の声が多い | デフォルト値（時給・手作業時間）の引き上げ検討 |
| 「中間値が欲しい」の声 | `INSOURCING_LEVELS` を 7 段階に拡張 |
| 「商談で根拠を問われない」 | 出典明記の効果は薄い。FAQ や README 経由の間接訴求に絞る |

### 4.2 結果フィードバック先
- 仕様書 `docs/spec/calculation-logic.md §3 / §8` に追記
- 必要なら別 Issue を起票（Issue #90 のフォローアップ）
