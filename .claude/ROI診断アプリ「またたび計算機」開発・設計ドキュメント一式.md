# **ROI診断アプリ「またたび計算機」開発・設計ドキュメント一式**

本ドキュメントは、株式会社ねこにまたたび（以下「当社」）が展開する次世代IT戦略の一環として開発する、ROI（投資対効果）診断アプリ「またたび計算機」の要件定義、技術要件、および詳細設計を網羅したものです。本アプリは、顧客企業の経営層に対し、ITコストの最適化とAI駆動開発への転換による具体的メリットを数値で提示することを目的とします。

## **1\. 要件定義書 (Requirements Definition)**

### **1.1. プロジェクトの背景と目的**

* **背景：** 多くの企業がベンダー依存によるITコストの高止まりと、開発スピードの鈍化に悩んでいる。  
* **目的：** 「止血（コスト削減）」「創出（業務自動化による利益）」「加速（機会損失の解消）」の3軸でROIを可視化し、当社の技術的優位性をエビデンスとして提示する。

### **1.2. ターゲットユーザー**

* 中小企業から中堅企業の経営者・役員。  
* ITコストの妥当性に疑問を感じている決裁権者。

### **1.3. 機能要件**

| 機能カテゴリ | 機能名 | 内容詳細   |
| :---- | :---- | :---- |
| 入力フォーム | 5つの心理的質問 | 月額ベンダー費用、改修費用、手作業人数、更新待ち期間、内製化状況を入力。 |
| 計算エンジン | ROI算出ロジック | 入力値に基づき、3年間の削減可能コストと年間創出利益をリアルタイム計算。 |
| 出力表示 | 診断結果ダッシュボード | 数値、グラフ、警告メッセージを視覚的に表示。 |
| 成果物生成 | PDFレポート出力 | 診断結果をA4サイズのPDFとして生成・ダウンロード。 |

### **1.4. 非機能要件**

* **レスポンシブ対応：** 商談の場でのタブレット・スマートフォン利用を想定し、全デバイスで快適に動作すること。  
* **パフォーマンス：** Cloudflare Pagesのエッジ配信を活用し、1秒以内のLCP（Largest Contentful Paint）を目指す。  
* **デザイン性：** 「プロフェッショナル × 猫モチーフ」のモダンで親しみやすいブランドイメージを維持。

## **2\. 技術要件書 (Technical Requirements)**

### **2.1. フロントエンド**

* **Framework:** Next.js 14+ (App Router採用)  
* **Language:** TypeScript (静的型付けによる堅牢性の確保)  
* **Styling:** Tailwind CSS (ユーティリティファーストによる高速開発)  
* **Icons:** Lucide-react (クリーンなベクターアイコン)

### **2.2. ライブラリ・API**

* **PDF生成:** jsPDF ＋ html2canvas (クライアントサイドでの即時生成)  
* **状態管理:** React Context API または標準の useState (小規模アプリのため軽量化を優先)

### **2.3. インフラ・デプロイ**

* **Hosting:** Cloudflare Pages (GitHub連携による自動デプロイ)  
* **Domain:** Cloudflareにて管理 (SSL/TLSの標準装備)

## **3\. 詳細設計書 (Detailed Design)**

### **3.1. 計算ロジック詳細**

診断の核となる計算ロジックを以下のように定義する。

| 項目 | 計算式 | 備考   |
| :---- | :---- | :---- |
| 3年間の止血（削減額） | (月額費用 × 36ヶ月) \+ (単発改修費 × 年3回 × 3年) | 内製化・AI化による外部流出コストの削減。 |
| 年間の利益創出額 | 手作業人数 × 1日2時間 × 月20日 × 12ヶ月 × 2,500円 | AI自動化による人的リソースの時給換算利益。 |
| スピード警告 | 更新待ちが「3ヶ月以上」の場合に発動 | 「CRITICAL OPPORTUNITY LOSS」と表示。 |

> 確定仕様は [`docs/spec/calculation-logic.md`](../docs/spec/calculation-logic.md) を参照（Issue #1 で確定）。固定値の扱い、内製化係数、最終計算式（擬似コード）、丸めルール等はそちらが正本。

### **3.2. コンポーネント構成**

1. **Layout:** ナビゲーション、フッター、猫モチーフの背景装飾。  
2. **InputForm:** Step-by-Step形式の質問コンポーネント。  
3. **ResultDashboard:** アニメーションを用いた数値カウンター、カード型UI。  
4. **PdfGenerator:** 隠しDOM要素（A4比率）を画像化しPDFに変換するロジック。

> 確定仕様は [`docs/spec/result-dashboard.md`](../docs/spec/result-dashboard.md) を参照（Issue #3 で確定）。`ResultDashboard` の指標カード構成、グラフ種別・ライブラリ選定、アニメーション方針、レスポンシブレイアウト、警告バナー UI 骨格、PDF 連携方針はそちらが正本。

### **3.3. デザインガイドライン**

* **ベースカラー:** \#F8FAFC (Slate 50\)  
* **アクセントカラー:** \#3B82F6 (Blue 500 / 知的)、\#F59E0B (Amber 500 / 猫の温かみ)  
* **タイポグラフィ:** Sans-serif (Inter または Noto Sans JP)
