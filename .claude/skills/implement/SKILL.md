## 手順

実装planファイルを受け取り、プランに沿って実装 → コミット → プッシュ → PR作成 → レビュー → レビューコメント投稿を一貫して行う。

### 1. ブランチのチェック

現在のブランチが `develop` でない場合は「⚠️ このスキルは develop ブランチ上でのみ実行できます。`git checkout develop` で切り替えてください。」と警告を出し、**処理を中断する**。

### 2. planファイルを読み込む

`$ARGUMENTS` からplanファイルのパスを取得し、内容を読み込む。ファイルが存在しない場合は「⚠️ 指定されたplanファイルが見つかりません。」と警告を出し、**処理を中断する**。

### 3. featureブランチを作成

`/branch` スキルと同じ形式でブランチを作成する：

1. planファイルの概要からブランチ名を英語ケバブケースに変換
2. `date +%Y%m%d` で日付を取得
3. `git checkout -b feature/<英語ケバブケース>_<YYYYMMDD>` でブランチを作成

### 4. 実装

planファイルの内容に従い、ステップバイステップで実装を進める。

- plan内の実装ステップを順番に実行する
- 必要に応じて `Grep` / `Glob` / `Agent（Explore）` で既存コードを調査する
- CLAUDE.md のルール（Sail経由のコマンド実行、Voltファンクショナルスタイル等）を遵守する
- 各ステップの実装が完了したら、そのステップで変更したファイルを確認する

### 5. コミット

`/commit` スキルと同じ手順でコミットする：

1. `git status` / `git diff` / `git diff --cached` / `git log --oneline -10` を並列で実行
2. 関連ファイルを `git add` でステージする（`.env` や認証情報ファイルは除外、`git add -A` は使わない）
3. 差分を分析して日本語のコミットメッセージを生成し、`git commit` を実行
4. 末尾に `Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>` を付与
5. pre-commit hook が失敗した場合は修正して新しいコミットを作成（`--amend` しない）

実装内容が大きい場合は、論理的な単位で複数回に分けてコミットする。

### 6. プッシュとPR作成

`/pr` スキルと同じ手順でPRを作成する：

1. ベースブランチは `develop` とする
2. `git push -u origin HEAD` でプッシュ
4. PRタイトル（日本語、70文字以内）と本文を生成し、`gh pr create --draft --base <base>` でドラフトPRを作成
5. planファイル内にイシュー番号（例: `#13`, `issue-13` 等）が含まれている場合、本文に `Closes #<番号>` を追加してマージ時にイシューが自動クローズされるようにする
6. 本文フォーマット：
   ```markdown
   ## Summary
   <箇条書きで変更内容を説明（日本語）>

   Closes #<イシュー番号>（該当する場合のみ）

   ## Test plan
   <テスト方法のチェックリスト>

   🤖 Generated with [Claude Code](https://claude.com/claude-code)
   ```

### 7. レビュー

作成したPRの差分を取得し、planファイルと照合しながら以下の観点でレビューする：

- **planとの整合性**: 実装内容がplanに沿っているか、漏れや逸脱がないか
- **コード品質**: 可読性、保守性、パフォーマンス
- **テストカバレッジ**: テストが十分に書かれているか
- **ドキュメント**: 必要なコメントやドキュメントが整備されているか
- **コード整理**: 不要なコード、デバッグ用コードが残っていないか
- **リファクタリング**: 改善の余地があるコード構造

レビューには `Agent（Plan）` を使用し、PR差分とplanファイルの内容を渡して分析させる。

### 8. レビューコメントを投稿

レビュー結果を以下の形式で整理し、PRにコメントとして投稿する：

```bash
gh pr comment <PR番号> --body "$(cat <<'EOF'
## Code Review

### 🔴 重要（Critical）
<修正が必要な問題。セキュリティ、バグ、データ損失のリスクがあるもの>
- 該当がなければ「なし」と記載

### 🟡 推奨（Recommended）
<改善を推奨する項目。コード品質、パフォーマンス、保守性に関するもの>
- 該当がなければ「なし」と記載

### 🟢 軽微（Minor）
<スタイル、命名、コメントなど軽微な改善提案>
- 該当がなければ「なし」と記載

---

### 総合評価
<以下のいずれかを表示>
- ✅ **LGTM** — planに沿った実装が完了しており、問題ありません。
- ⚠️ **修正が必要** — 上記の指摘事項を修正してください。

🤖 Generated with [Claude Code](https://claude.com/claude-code)
EOF
)"
```

### 9. 結果を表示

- 作成したPRのURL
- レビュー結果の概要（重要/推奨/軽微の件数と総合評価）
