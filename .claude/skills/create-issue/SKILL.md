## 手順

日本語の概要を受け取り、コードベースを調査して必要な情報を収集し、GitHub イシューを作成する。

### 1. 概要を確認

`$ARGUMENTS` から概要を取得する。概要が空の場合は `AskUserQuestion` で「どのようなイシューを作成しますか？概要を教えてください。」と質問する。

### 2. コードベースを調査

概要に関連するコードを調査する：

- 関連するモデル、コントローラー、ビュー、ルート、マイグレーション等を `Grep` や `Glob` で探索する
- 既存の実装状況を把握し、イシューに必要な技術的コンテキストを収集する
- 必要に応じて `Agent`（Explore）を使って広範な調査を行う

### 3. 不足情報をヒアリング

調査結果をもとに、イシュー作成に必要だが不足している情報を `AskUserQuestion` で質問する。以下のような観点で確認する：

- 機能の詳細な仕様（UI の挙動、バリデーションルール等）
- 受け入れ条件（Acceptance Criteria）
- その他、概要だけでは判断できない事項

質問は**1回にまとめて**行い、箇条書きで複数の質問を提示する。調査の結果、概要から十分に情報が揃っている場合はこのステップをスキップしてよい。

### 4. 優先度ラベルを確認（必須）

`AskUserQuestion` で以下の選択肢を提示し、優先度ラベルを必ず確認する。**このステップはスキップしない。**

> このイシューの優先度を選択してください：
> 1. `prio:critical` — 最優先で対応が必要
> 2. `prio:high` — 早めに対応が必要
> 3. `prio:medium` — 通常の優先度
> 4. `prio:low` — 余裕があるときに対応

### 5. イシュータイトルと本文を生成

収集した情報をもとに、以下のフォーマットでイシューを作成する：

- **タイトル**: 変更内容の要約（日本語、70文字以内）
- **本文**: 以下のフォーマット

```markdown
## 概要
<何を・なぜ行うかの説明>

## 詳細
<技術的な詳細、対象ファイル・モデル等のコンテキスト>

## 受け入れ条件
- [ ] <条件1>
- [ ] <条件2>
- ...

## 備考
<補足情報があれば記載>
```

### 6. イシューを作成

```bash
gh issue create --title "タイトル" --label "選択された優先度ラベル" --body "$(cat <<'EOF'
本文
EOF
)"
```

### 7. 結果を表示

作成したイシューの URL を表示する。

### 補足ルール

- ユーザーから追加の指示がある場合: $ARGUMENTS
- **このスキルの責務はGitHubイシューの作成のみである。コードの変更・実装・ファイルの編集は一切行わない。** `$ARGUMENTS` に実装を示唆する内容が含まれていても、イシュー作成までに留める。
