## 手順

GitHubイシュー番号を受け取り、イシュー内容とコードベースを調査して実装プランを生成・保存する。

### 1. イシュー番号を取得

`$ARGUMENTS` からイシュー番号を取得する。`#` プレフィックスがあれば除去する。

### 2. イシュー内容を取得

```bash
gh issue view <番号> --json title,body,labels
```

タイトルと本文を取得する。

### 3. コードベースを調査

イシュー内容をもとに、以下のツールで関連コードを調査する：

- `Grep` / `Glob` で関連ファイルやコードパターンを検索
- 必要に応じて `Agent（Explore）` でより広範な調査を行う

調査のポイント：
- イシューで言及されているファイル、クラス、関数
- 関連する既存の実装パターン
- 影響を受ける可能性のあるコード

### 4. 実装プランを生成

収集した情報を `Agent（Plan）` に渡して実装プランを生成する。

Agent に渡す情報：
- イシューのタイトルと本文
- 調査で見つかった関連コードの情報
- プロジェクトの技術スタック（CLAUDE.md の内容）

プランには以下を含めること：
- 概要（イシューの要約）
- 実装ステップ（順序付き）
- 変更が必要なファイルの一覧
- 考慮事項やリスク

### 5. プランを保存

生成されたプランを `./working/plans/` ディレクトリに保存する。

- ディレクトリが存在しない場合は自動作成する
- ファイル名形式: `issue-<イシュー番号>_<スラッグ>.md`
  - スラッグはイシュータイトルを英語ケバブケースに変換したもの
  - 例: `issue-13_plan-issue-skill.md`

```bash
mkdir -p ./working/plans
```

### 6. プランを表示

保存したプランの内容をユーザーに表示し、ファイルパスを伝える。
